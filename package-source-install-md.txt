# INSTALL.md — mycmux セットアップ runbook

Claude Code はこの手順を上から順に実行してください。途中で失敗したら即停止し、
エラー全文を貼って相談してください。動作環境は Windows 10 / 11 (x64) のみです。

---

## Step 1. 前提ツールの確認

以下をそれぞれ実行し、バージョン要件を満たすか確認する。

### 1-1. Node.js (20 以上)

```powershell
node --version
```

なければ: https://nodejs.org/ja/download から LTS (20 以上) を入れてもらう。
ユーザーに許可を得てから自動インストールは一切しない。案内だけでよい。

### 1-2. Rust (stable, MSVC toolchain)

```powershell
rustc --version
rustup show active-toolchain
```

`rustc` が無ければ https://rustup.rs/ から `rustup-init.exe` を入れるよう案内。
active-toolchain が `stable-x86_64-pc-windows-msvc` であること。異なる場合は
`rustup default stable-x86_64-pc-windows-msvc` を提案。

### 1-3. Microsoft C++ Build Tools 2022

以下のいずれかに `vcvarsall.bat` が存在すれば OK。

- `C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvarsall.bat`
- `C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvarsall.bat`
- `C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat`
- `C:\Program Files\Microsoft Visual Studio\2022\Professional\VC\Auxiliary\Build\vcvarsall.bat`
- `C:\Program Files\Microsoft Visual Studio\2022\Enterprise\VC\Auxiliary\Build\vcvarsall.bat`

無ければ「Build Tools for Visual Studio 2022」を
https://visualstudio.microsoft.com/visual-cpp-build-tools/ から入れ、
インストーラで **「C++ によるデスクトップ開発」ワークロード** を選択するよう案内。

### 1-4. WebView2 Runtime

Windows 11 には標準搭載。Windows 10 で無い場合のみ、後でアプリが起動しない症状が
出たら https://developer.microsoft.com/ja-jp/microsoft-edge/webview2 から Evergreen を入れる。
(通常は tauri ビルドが同梱するので不要)

---

## Step 2. 依存関係のインストール

プロジェクトルート (この `INSTALL.md` があるフォルダ) で実行:

```powershell
npm install
```

数分かかる。`npm warn` は無視してよい。`npm error` が出たら停止して相談。

---

## Step 3. ビルド

```powershell
powershell -ExecutionPolicy Bypass -File .\build-full.ps1
```

`build-full.ps1` が MSVC 環境を自動検出して `npm run tauri build` を実行する。
Rust の初回コンパイルは 5〜15 分。`Finished release` が出たら成功。

成果物:

- `src-tauri\target\release\mycmux.exe` — 本体
- `src-tauri\target\release\bundle\nsis\mycmux_{VERSION}_x64-setup.exe` — NSIS インストーラ (任意で使う)

---

## Step 4. インストール先を決めてコピー

ユーザーに置き場所を確認する。デフォルトは `%USERPROFILE%\mycmux-app\`。

```powershell
$dest = "$env:USERPROFILE\mycmux-app"
New-Item -ItemType Directory -Path $dest -Force | Out-Null
Copy-Item ".\src-tauri\target\release\mycmux.exe" -Destination "$dest\mycmux.exe" -Force
```

既に `mycmux.exe` が起動中だと上書きに失敗する。エラーが出たら起動中プロセスを
終了してもらってから再実行する。

---

## Step 5. デスクトップショートカット (任意)

ユーザーが望む場合のみ実行:

```powershell
$wshell = New-Object -ComObject WScript.Shell
$link = $wshell.CreateShortcut("$env:USERPROFILE\Desktop\mycmux.lnk")
$link.TargetPath = "$env:USERPROFILE\mycmux-app\mycmux.exe"
$link.IconLocation = "$env:USERPROFILE\mycmux-app\mycmux.exe,0"
$link.Save()
```

---

## Step 6. 起動確認

```powershell
Start-Process "$env:USERPROFILE\mycmux-app\mycmux.exe"
```

初回起動時に Windows SmartScreen が警告を出す (署名無しビルドのため)。
ユーザーに「詳細情報」→「実行」で進むよう案内する。

起動したらウィンドウが出てシェルが 1 ペイン開くはず。ここで手順完了を報告。

---

## トラブルシューティング

### npm install で `node-gyp` / `python` エラー

Node.js の付属 Python で足りない場合がある。Python 3.11 以上を入れるか、
`npm install --no-optional` を試す。

### tauri build が link.exe エラー

Git Bash がインストールされていると `/usr/bin/link.exe` (GNU link) が優先されて
MSVC link.exe と衝突することがある。`build-full.ps1` 経由で実行すれば vcvarsall が
MSVC link を先頭に置くので解消する。PowerShell を新規に開き直して再実行する。

### `error: linker not found` / `cl.exe not found`

MSVC Build Tools が入っていないか、`C++ によるデスクトップ開発` ワークロードが未選択。
Visual Studio Installer を開いて当該ワークロードを追加する。

### 起動直後にウィンドウが一瞬だけ出て消える

WebView2 Runtime が無い。上記 Step 1-4 を参照。

### PTY が動かない / シェルが検出されない

Windows は Git Bash を優先検出する。Git for Windows が入っていれば自動で使われる。
そうでなければ PowerShell / cmd にフォールバック。Git Bash を入れるのが一番安定。

---

## 完了基準

- `mycmux.exe` が起動してウィンドウが出る
- 1 つのシェルペインが自動で開いてプロンプトが出る
- `Ctrl+Shift+→` で右ペイン分割、`Ctrl+Shift+W` でペイン閉じができる

この 3 点が動いたらインストール完了。ユーザーにショートカットの場所と
起動方法を伝えて終了する。
